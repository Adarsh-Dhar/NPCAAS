You are absolutely right. A dashboard layout feels like a decentralized finance app, not a real video game. To make players feel like they are actually inhabiting a cyberpunk world, we must abandon the standard "web page" layout and move to a **Canvas-based Game Loop**. 

Instead of clicking tabs to talk to characters, the player needs to spawn into a 2D or 3D world, walk their avatar over to an NPC, and trigger the chat interface via proximity (e.g., pressing "E" to talk).

Here is the architectural shift and the exact plan to build an actual web game in Next.js that integrates with your SDK.

### The Hybrid Architecture: Canvas + React Overlay
You will use a "Hybrid Architecture." You do not build UI elements (like chat boxes or text inputs) inside the game engine itself because game engines are terrible at handling text inputs and web requests. Instead:
1. **The Game Layer (Canvas):** Handles movement, rendering the map, collision, and physics.
2. **The UI Layer (React):** Standard Next.js components (your existing chat and wallet UI) that float *transparently on top* of the game canvas, hiding and showing based on what is happening in the game.

### The Implementation Plan

To build this in your Next.js environment, you need a lightweight game engine. The best tool for 2D web games in React is **Phaser.js** (specifically using the `@phaserjs/react` wrapper).

Here is how you structure it step-by-step:

#### Step 1: Install the Game Engine
Run this in your Next.js frontend directory:
```bash
pnpm add phaser @phaserjs/react
```

#### Step 2: Create the Game Scene (The Canvas)
You will create a Phaser Scene that handles the actual game world. This file does not care about your AI or Web3 logic; it only cares about pixels and coordinates.

```javascript
// game/scenes/MainScene.js
import Phaser from 'phaser';

export class MainScene extends Phaser.Scene {
  constructor() {
    super('MainScene');
  }

  preload() {
    // Load sprites (e.g., a cyberpunk city tileset, player sprite, NPC sprites)
    this.load.image('player', '/sprites/player.png');
    this.load.image('scrap_npc', '/sprites/scrap.png');
  }

  create() {
    this.player = this.physics.add.sprite(400, 300, 'player');
    this.scrap = this.physics.add.staticSprite(200, 200, 'scrap_npc');
    
    this.cursors = this.input.keyboard.createCursorKeys();
    this.interactKey = this.input.keyboard.addKey(Phaser.Input.Keyboard.KeyCodes.E);
  }

  update() {
    // 1. Handle Player Movement (WASD)
    this.player.setVelocity(0);
    if (this.cursors.left.isDown) this.player.setVelocityX(-160);
    if (this.cursors.right.isDown) this.player.setVelocityX(160);
    if (this.cursors.up.isDown) this.player.setVelocityY(-160);
    if (this.cursors.down.isDown) this.player.setVelocityY(160);

    // 2. Proximity Checking
    const distanceToScrap = Phaser.Math.Distance.BetweenPoints(this.player, this.scrap);
    
    // 3. Emit event to React if close enough and pressing 'E'
    if (distanceToScrap < 50 && Phaser.Input.Keyboard.JustDown(this.interactKey)) {
        // Emit an event that the React layer is listening to
        window.dispatchEvent(new CustomEvent('OPEN_CHAT', { detail: { npcId: 'scrap' } }));
    }
  }
}
```

#### Step 3: The React Overlay Wrapper
Now, you wrap the Phaser game in a React component and position your Chat UI absolutely over it. When Phaser emits the `OPEN_CHAT` event, React catches it and renders the UI.

```tsx
// app/game/page.tsx
'use client'
import { useEffect, useState } from 'react';
import { App } from '@/game/App'; // Your Phaser initialization
import ChatWindow from '@/components/ChatWindow'; // Your actual SDK chat component
import PlayerRig from '@/components/PlayerRig'; // Your Web3 balances

export default function GamePage() {
  const [activeNpc, setActiveNpc] = useState<string | null>(null);

  useEffect(() => {
    // Listen for the game engine telling React to open the UI
    const handleOpenChat = (event: CustomEvent) => {
      setActiveNpc(event.detail.npcId);
      // Optional: Dispatch event back to Phaser to pause movement
    };

    window.addEventListener('OPEN_CHAT', handleOpenChat as EventListener);
    return () => window.removeEventListener('OPEN_CHAT', handleOpenChat as EventListener);
  }, []);

  return (
    <div className="relative w-screen h-screen bg-black overflow-hidden">
      
      {/* LAYER 1: The Game Canvas (Z-Index 0) */}
      <div id="phaser-game-container" className="absolute inset-0 z-0" />
      
      {/* LAYER 2: The Player UI (Z-Index 10) */}
      <div className="absolute top-4 right-4 z-10 pointer-events-none">
         {/* Use pointer-events-auto inside the component so clicks register */}
        <PlayerRig /> 
      </div>

      {/* LAYER 3: The Chat Overlay (Z-Index 20) */}
      {activeNpc && (
        <div className="absolute left-4 bottom-4 w-1/3 h-2/3 z-20">
          <ChatWindow 
            npcId={activeNpc} 
            onClose={() => setActiveNpc(null)} 
          />
        </div>
      )}

    </div>
  );
}
```

### Why this is the right way to do it:
* **The Game feels like a Game:** The player is physically exploring a space, discovering NPCs, and moving around.
* **Your SDK stays perfectly intact:** You don't have to rewrite your Next.js API routes (`/api/chat`, `/api/npcs/[id]/wallet`). You are simply swapping out *how* the chat window is triggered (keyboard collision event vs. clicking a dashboard tab).
* **Easy to Expand:** If you want The Enforcer NPC to walk around the map, you program his movement logic in `MainScene.js`. If you want him to buy items, you have the Phaser scene hit your Next.js `/api/npcs/[id]/wallet/interact` route when he reaches a specific coordinate.
